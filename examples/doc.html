Special Variables

oMEOB.set("MEOB.Template","index3-t.html");

the name of the template file which contains layout and placeholder information

oMEOB.set("MEOB.Static","index.props");
 the name of the file that contains content that does not change - static content
oMEOB.set("MEOB.WriteTo","MEOB_OUT");
 the Name of the DIV to which the HTML should be written to. This should match the id of the DIV above (<DIV id=MEOB_OUT></div>)
oMEOB.set("MEOB.StyleSheet","stylesheet.css");
 dynamic stylesheet that should be used (not implemented)
oMEOB.set("MEOB.mergedelim.begin","[[");
 sequence of characters used to identify the start of a placeholder found in the template file
oMEOB.set("MEOB.mergedelim.end","]]"); sequence of characters used to identify the end of a placeholder found in the template file.

Template Files

A template file is a valid HTML file that has placeholders embedded within it. Javascript code should be placed in separate js files that are referenced by the template.

At present placeholders of the same kind can not be nested.

placeholders

here are the possible kinds of placeholders

[[var]]

A simple variable substitution. MEOB will look through the IDs of the static and dynamic variables and replace this tag with the value if found.
[[ifne.var]]
HTML fragment
[[end ifne.var]]
 displays the HTML fragment if the variable is not found
[[if.var]]
HTML fragment
[[end if.var]]
 displays the HTML fragment if the variable is found
[[if.var1 eq var2]]
HTML fragment
[[end if.var]]
 displays the HTML fragment if the variable1 is equal to variable 2
[[if.var1 ne var2]]
HTML fragment
[[end if.var]]
 displays the HTML fragment if the variable1 is not equal to variable 2
[[if.var1 gt var2]]
HTML fragment
[[end if.var]]
 displays the HTML fragment if the variable1 is greater than variable 2
[[if.var1 lt var2]]
HTML fragment
[[end if.var]]		 displays the HTML fragment if the variable1 is less than variable 2
[[repeat.var loopbreak n]]
HTML fragment
[[end repeat.var]]		 Repeats the HTML fragment for as many loop control variables of the form var.[n] are defined, where n is a number from 1 upwards.
[[loop]] When within a repeat block is replaced by the value of var.[n]
[[loop.myvalue]] When within a repeat block is replaced by the value of var.[n].myvalue
[[loopkey]] used within repeat blocks for debugging purposes to show the name of the current loop control variable.
[[if.loopbreaknow eq y]]...[[end if.loopbreaknow]] used within repeat, process contents when loop index is 0, either at start of loop, or when loop breaks due to loopbreak
[[if.loopbreaknext eq y]]...[[end if.loopbreaknext]] used within repeat, process contents when loop index is going to be 0, either before loop breaks due to loopbreak or at last item.
[[loopprevious]] the previous loop item if it exists
[[loopnext]] the next loop item if it exists
[[include.myfilename]] inserts the contents of another template

Static Variables

Static variables are kept in a properties file which is a well formed XML file with Key Value pairs.
Here is an example of a properties file:

<?xml version="1.0" encoding="UTF-8" ?>
<MEOB>
  
<div id="document.title">if example</div>
<div id="control1">yes</div>
<div id="value1">This text is visible and should be in red</div>
<div id="value2">this text should not be visible</div>
<div id="blurb1">This example shows how a fragment of HTML is displayed when a control value is present</div>
<div id="blurb2">there should be a table with a red background above, but nothing below</div>
<div id="control3">12</div>
<div id="userid">12</div>
 key value pairs are defined here and are substituted into the placeholders in the template by the MEOB code.
</MEOB>
